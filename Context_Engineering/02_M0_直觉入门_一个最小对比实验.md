# 一个最小对比实验

**一句话：** 同一个问题，换不同的 context，输出天差地别——这个实验能让你直观感受到 context 的决定性作用。

## 是什么

下面是一个你可以立刻动手试的实验。任务很简单：让模型回答一个电商客服问题。我们只改变 context，观察输出如何变化。

**用户问题（始终不变）：**
> 你们的退货政策是什么？

---

### 实验 A：裸奔——没有任何额外 context

```
用户：你们的退货政策是什么？
```

**典型输出：**
模型会给一个泛泛的、看起来合理但完全编造的回答：
> "一般来说，大多数电商平台支持7天无理由退货。您需要保证商品完好，附上发票……"

这些信息可能对也可能错——模型在"猜"，因为它没有任何关于"你们"是谁的信息。

---

### 实验 B：加入身份——给一个 system prompt

```
System：你是"鲜果优选"的客服助手，一家专注生鲜水果的电商平台。

用户：你们的退货政策是什么？
```

**典型输出：**
> "鲜果优选支持收货后48小时内申请退货退款。由于生鲜商品的特殊性，请您在签收时检查商品状态……"

好了一些——模型知道自己是谁了，回答开始有针对性。但退货政策的细节（48小时？签收时检查？）仍然是模型编的，因为你没有给它真正的政策文档。

---

### 实验 C：注入真实知识——把政策文档放进 context

```
System：你是"鲜果优选"的客服助手。回答用户问题时，严格依据下方的政策文档，不要编造政策中没有的内容。

## 退货政策
1. 签收后24小时内可申请退货
2. 需提供开箱视频作为凭证
3. 以下情况不支持退货：
   - 已切开的水果
   - 无开箱视频
   - 超过24小时
4. 退款将在审核通过后3个工作日内原路返回

用户：你们的退货政策是什么？
```

**典型输出：**
> "鲜果优选的退货政策如下：您需要在签收后24小时内提出申请，并提供开箱视频作为凭证。请注意，已切开的水果、没有开箱视频、或超过24小时的情况不支持退货。退款会在审核通过后3个工作日内原路返回到您的账户。"

准确、具体、没有编造。

---

## 为什么重要

三次实验，同一个模型，同一个问题，三种完全不同的输出质量：

| 实验 | Context 内容 | 输出质量 |
|------|-------------|----------|
| A | 只有用户问题 | 泛泛而谈，可能编造 |
| B | + 身份 system prompt | 有角色感，但细节仍是编造 |
| C | + 身份 + 真实政策文档 | 准确、可靠、可用于生产 |

这三次实验揭示的核心规律是：

- **模型的输出质量上限由 context 决定。** 模型只能用你给它的材料来作答。给垃圾 context，再强的模型也只能输出垃圾（Garbage In, Garbage Out）。
- **每一层 context 解决一类问题。** 身份解决"我是谁"，知识文档解决"事实是什么"，规则约束解决"该怎么说"。缺任何一层，输出就会在那个维度上失控。
- **从 A 到 C 的改进不需要更好的模型。** 完全是 context 设计的改进。这就是 Context Engineering 的核心价值——用系统化的方法设计 context，而不是碰运气。

## 设计启示

- **永远从"模型此刻能看到什么"开始诊断问题。** 当输出不符合预期时，不要急着换模型或调参数。把 context 打印出来看一遍，答案往往就在那里。
- **分层思考 context。** 不同层次的信息解决不同的问题：身份定位、事实依据、行为规则、输出格式。有意识地为每一层提供信息，而不是把所有东西混在一起。
- **可以把这个实验当作日常调试工具。** 遇到输出问题时，退回到最简 context，然后逐层添加，观察哪一层的添加引发了变化——这就是最小对比实验法。

## 常见误区

**误区一："实验 C 的效果好只是因为 prompt 写得更长"**
不是长度的问题，而是相关性的问题。如果你在实验 C 中放入一万字的无关内容（比如公司的创业历史），效果反而可能比实验 B 更差。关键不在于 context 多长，而在于 context 里有没有回答这个问题所需要的信息。

**误区二："实际项目中手动把知识粘贴进 prompt 就行了"**
这在玩具示例中可以，但在真实系统中不可行——你不知道用户会问什么问题，不可能提前粘贴所有可能用到的知识。这就是为什么需要 RAG（检索增强生成）、动态 context 组装等技术，这些是 M2 的核心内容。

**误区三："只要 context 里有正确信息，模型就一定会用"**
不一定。信息的位置、格式、表达方式都会影响模型是否能有效利用它。比如同样的政策文档，结构化的列表比一大段自然语言段落更容易被模型准确提取。这涉及到 context 质量的问题，M1 会深入讨论。
