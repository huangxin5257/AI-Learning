**调试方法论**

1. 最小复现 — 剥离无关上下文，找到能稳定触发问题的最简 Prompt。不会最小复现，调试就变成大海捞针，改了十处不知道哪处起了作用。
2. 隔离变量 — 每次只改一处，确保改动与效果之间有因果关系。同时改多处是最常见的调试陷阱，导致你永远不知道是什么让它变好或变坏。
3. 消融测试 — 逐一移除 Prompt 组件，量化每个部分的实际贡献。很多 Prompt 里有大量"感觉有用"的内容，消融测试帮你发现哪些其实是冗余甚至有害的。
4. 随机性区分 — 调高 temperature 多次采样，判断错误是偶发还是系统性。偶发错误和系统性错误的修复方向完全不同，不区分就会走错路。
5. 多模型交叉验证 — 同一 Prompt 在不同模型上测试，区分 Prompt 问题与模型能力边界。避免花大量时间优化一个 Prompt，其实问题根本出在模型能力上而非写法上。
