**评估体系**

1. Eval Set 构建 — 准备覆盖正常、边界、对抗三类场景的测试用例集。没有固定测试集，每次评估的标准都不一样，迭代就没有可比性。
2. 评分维度设计 — 预先定义准确性、格式、风格等评分维度，避免主观随意打分。维度不清晰时，"感觉好一点"会成为唯一判据，无法支撑团队协作和长期迭代。
3. LLM-as-Judge — 用模型对输出打分，需注意位置偏差、自我偏好等已知偏差。不了解其偏差就直接信任评分结果，等于用一把不准的尺子量东西还以为结果是精确的。
4. 人工抽检 — 对模型评分结果按比例人工复核，校准自动评分的可信度。自动评分会系统性地偏向某类输出，不抽检就无法发现评估体系本身已经失准。
5. 回归测试 — 每次修改 Prompt 前在固定 Eval Set 上跑一遍，防止改出新问题。Prompt 改动几乎必然有副作用，不回归就会积累隐性退步，直到线上出问题才发现。
