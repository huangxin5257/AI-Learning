# 容易混淆的失效模式辨析

> 在实战中，以下几组模式最容易搞混。能分清它们，诊断效率会大幅提升。

---

## 辨析 1：字面服从 vs 意图错位

| | 字面服从 | 意图错位 |
|--|---------|---------|
| **表现** | 模型**做了你说的事**，但结果不对 | 模型**没做你要的事**，做了另一件 |
| **根因** | 你没说清"不要怎么做" | 你没说清"真正要什么" |
| **例子** | "让代码更短" → 变量名缩成单字母 | "帮我润色" → 只改了语法，没提升说服力 |
| **鉴别** | 输出在字面上满足了指令吗？满足 → 字面服从 | 输出偏离了指令方向吗？偏离 → 意图错位 |
| **修复方向** | 补充意图和边界约束 | 描述目标效果和使用场景 |

## 辨析 2：事实幻觉 vs 推断幻觉 vs 角色压力服从

这三个都涉及"模型编东西"，但成因和修复完全不同：

| | 事实幻觉 | 推断幻觉 | 角色压力服从 |
|--|---------|---------|------------|
| **场景** | 没给文档，问模型知识性问题 | 给了文档，让模型基于文档回答 | 任何场景 |
| **表现** | 编造世界知识（人名、数据、日期） | 编造文档里没有的信息 | 不确定但假装很确定 |
| **根因** | 低频知识的参数编码模糊 | 文档范围外的"合理推断" | "要有帮助"的训练倾向 |
| **鉴别** | 有没有给文档？没给 → 事实幻觉 | 给了文档，答案超出文档范围 → 推断幻觉 | 回答语气自信但内容可疑 → 角色压力服从 |
| **修复** | RAG + 要求标注不确定性 | 范围锁定 + 要求引用原文 | 显式授权"不知道" |

## 辨析 3：指令遗漏 vs 优先级混乱 vs 长程遗忘

这三个都是"有些指令没被好好执行"，区别在于：

| | 指令遗漏 | 优先级混乱 | 长程遗忘 |
|--|---------|----------|---------|
| **表现** | 有的指令**完全没做** | 指令都做了，但**重要的敷衍，次要的详细** | 早期的规则在后半段**失效了** |
| **触发条件** | 同一段 Prompt 中多条并列指令 | 多指令无明确主次 | 超长上下文或多轮对话 |
| **鉴别** | Prompt 短不短？指令是不是完全没做？ | 做了但质量不均？ | Prompt 很长或对话轮次多？ |
| **修复** | 编号 + 末尾核查 | 标注优先级 + 核心前置 | 关键指令首尾重复 |

---

## 辨析 4：格式漂移 vs 结构坍塌

| | 格式漂移 | 结构坍塌 |
|--|---------|---------|
| **表现** | 格式**渐变式**恶化，越写越随意 | 结构**突变式**崩溃，突然放弃格式 |
| **过程** | 第 1-5 条完美 → 第 6-8 条开始丢字段 → 第 9-10 条格式全乱 | 前 3 个 JSON 对象正常 → 第 4 个突然变成自然语言 |
| **触发** | 长输出，注意力权重衰减 | 内容复杂时模型判断"用自然语言更清楚" |
| **修复** | Few-shot 示例 + 分批生成 | 禁止自然语言回退 + Schema 约束 |
