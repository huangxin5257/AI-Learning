## 5. AI 辅助开发

| 知识点                   | 是什么                                                       | 为什么学                                             |
| ------------------------ | ------------------------------------------------------------ | ---------------------------------------------------- |
| AI编辑器深度使用         | Cursor / Windsurf 等工具的核心功能与工作流（Rules、Composer、多文件编辑） | 工具用对了效率是10倍，用错了是噪音                   |
| Claude Code / CLI Agent  | 在终端中以对话方式完成代码任务的命令行工具                   | 适合无界面场景和自动化脚本，与编辑器形成互补         |
| 上下文投喂策略           | 向AI提供哪些文件、架构文档、约束信息以提升代码质量           | AI生成代码的质量上限由你给的上下文决定               |
| 代码审查与验收           | 如何高效review AI生成代码，识别逻辑漏洞和安全隐患            | AI会自信地写出错误代码，审查能力是核心护城河         |
| 任务拆解与指令设计       | 将大型开发任务分解为AI可执行的原子步骤                       | 大任务直接扔给AI成功率低，拆解能力决定复杂项目落地率 |
| 测试驱动的AI开发         | 先写测试用例再让AI补实现，用测试约束AI输出                   | 大幅降低AI生成代码的验收成本，提高可靠性             |
| 多智能体协作编程         | 多个AI角色分工（架构师/编码/审查）协同完成项目               | 单Agent上下文有限，多Agent分工是复杂项目的必然趋势   |
| .cursorrules / CLAUDE.md | 项目级AI行为约束文件，定义代码风格、禁止模式、项目背景       | 让AI在整个项目生命周期内保持一致性，减少重复纠偏     |
