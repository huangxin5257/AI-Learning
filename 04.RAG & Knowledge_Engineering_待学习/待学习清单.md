## 4. RAG / 知识工程

| 知识点                    | 是什么                                                       | 为什么学                                             |
| ------------------------- | ------------------------------------------------------------ | ---------------------------------------------------- |
| Embedding 原理            | 将文本转换为向量空间中可计算距离的数值表示                   | 理解语义检索的底层机制，选型时不会盲目               |
| 文档分块策略（Chunking）  | 将长文档切割成适合检索的片段的方法论                         | 分块质量直接决定检索准确率，是RAG调优第一关          |
| 向量数据库                | 专门存储和检索高维向量的数据库系统（如 Pinecone、Weaviate、pgvector） | AI应用的核心基础设施，必须会选型和使用               |
| 相似度检索算法            | ANN近似最近邻等高效向量匹配算法                              | 理解检索速度与精度的权衡，避免线上性能问题           |
| 混合检索（Hybrid Search） | 将向量语义检索与关键词检索（BM25）结合使用                   | 纯向量检索对精确词匹配天然偏弱，混合检索是生产级标配 |
| Reranking 重排序          | 对初步检索结果用更精细的模型二次排序                         | 低成本显著提升最终答案质量，效果/成本比极高          |
| RAG Pipeline 架构         | 从文档入库到最终生成的完整数据流设计                         | 理解各环节依赖关系，才能系统性排查质量问题           |
| 知识库更新策略            | 文档增删改时如何保持索引与源数据一致                         | 生产环境知识库不是静态的，更新机制设计不当会导致幻觉 |
| 评估指标（RAG Eval）      | 检索召回率、答案忠实度、相关性等量化指标                     | 没有评估就没有迭代方向，是RAG工程化的必备工具        |
| GraphRAG                  | 将知识图谱引入检索路径，处理多跳推理问题                     | 弥补向量检索对结构化关系推理的天然短板               |
